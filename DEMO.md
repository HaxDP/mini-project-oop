# DEMO

Цей сценарій демонстрації розрахований на короткий захист без зайвих переходів. Спочатку збери й запусти застосунок:

```bash
cmake -S . -B build
cmake --build build --config Debug
./build/Debug/cinema_app.exe
```

Потім послідовно покажи три ключові моменти. Спершу створи фільм, зал і сеанс, щоб комісія побачила стартову підготовку даних. Далі забронюй місце і одразу викупи бронювання: це закриває основний бізнес-потік від замовлення до квитка. Після цього спеціально повтори бронювання того самого місця або введи недостатню оплату, щоб показати контроль помилок.

На завершення закрий програму, запусти її ще раз і відкрий довідники або аналітику. Так добре видно, що дані не губляться між сесіями і persistence працює коректно.
