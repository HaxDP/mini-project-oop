# DEVELOPER GUIDE

Проєкт зібрано як класичний приклад шарової архітектури для курсу ООП. Domain описує сутності й правила, application тримає use case-логіку, infrastructure відповідає за файлові репозиторії, а presentation надає консольний інтерфейс.

Цей поділ зроблений не формально, а для практичної розширюваності. Якщо потрібно змінити спосіб збереження, достатньо дати іншу реалізацію репозиторіїв. Якщо треба інша формула ціни, додається нова стратегія без переписування сервісу.

Для локальної перевірки використовуй стандартний цикл:

```bash
cmake -S . -B build
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

CI-конвеєр знаходиться у .github/workflows/ci.yml і автоматизує збірку, тести та coverage gate. Це дозволяє тримати якість стабільною навіть при частих змінах.
