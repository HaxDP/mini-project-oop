# Iteration 1

На першій ітерації було важливо не розпорошуватись, а закласти міцний фундамент. Тому основна робота пішла в опис проблеми, базову доменну модель і загальний каркас архітектури. Паралельно з'явився перший вертикальний зріз, який уже можна було запустити.

Результат ітерації: є структура шарів, зрозумілі межі відповідальності і перший робочий функціональний шлях, від якого далі розвивався весь проєкт.
