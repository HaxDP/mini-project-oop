# Extension Plan

Наступний логічний крок для цього проєкту - перейти від файлового збереження до SQLite, щоб підняти надійність і спростити складні запити. Архітектура вже підготовлена до такого переходу, бо application-шар залежить від контрактів, а не від конкретних файлів.

Після цього планується винести зовнішній API-шар, щоб система могла працювати не лише як консольний застосунок. У перспективі це відкриває шлях до web-клієнта та ролей користувачів.

Критерієм успіху розширення буде збереження поточних контрактів use case-сервісу, проходження існуючих тестів без деградації і підтримка або підвищення покриття.
