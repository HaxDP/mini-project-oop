# CinemaOOP

CinemaOOP - це навчальний проєкт з ООП на C++20 про роботу невеликого кінотеатру.
Логіка проста: адміністратор додає фільми, зали та сеанси, а касир бронює місця, приймає оплату і оформлює квитки.

Головна ідея цього проєкту - показати не просто CRUD, а нормальну бізнес-логіку:
перевірку валідності місця, захист від подвійного бронювання, динамічну ціну та керування станом бронювання.

Архітектура шарова: `domain`, `application`, `infrastructure`, `presentation`.
Такий поділ вийшов практичним: код легше пояснювати на захисті і простіше розширювати далі.

Дані зберігаються у каталозі `storage`, тому після перезапуску нічого не губиться.
Для цього використовуються окремі файли для фільмів, залів, сеансів, бронювань і квитків.

## Як запустити

```bash
cmake -S . -B build
cmake --build build --config Debug
./build/Debug/cinema_app.exe
```

## Як запустити тести

```bash
ctest --test-dir build -C Debug --output-on-failure
```

## Що покрито в роботі

У проєкті є інкапсуляція, інтерфейси, узагальнення через `Result<T>`, робота з колекціями,
обробка помилок, SOLID-підхід, патерни Repository/Strategy/Factory, UML-артефакти й тестова документація.
Всі матеріали знаходяться в корені репозиторію та в каталозі `docs`.
